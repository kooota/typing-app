# 5歳向けローマ字タイピングWebアプリ 初版計画

## Summary
初版は、5歳児が Chrome で遊びながらローマ字入力とPC操作に慣れるための、Web公開前提のミニゲーム型アプリとして作る。

体験は「短いステージ制」「キーボード中心」「端末内に進捗保存」で統一し、最初の教材は `母音 -> か行 -> さ行` の順で段階的に解放する。

追加要件として、画面上に英字キー中心の仮想キーボードを表示し、次に押すべきキーを色で強調する。キーガイドは保護者が ON/OFF を切り替えられる。

このアプリはローマ字学習だけでなく、`ひらがな -> 英字キー` の対応を覚えることも副目的に含める。初版ではキーガイドを既定で ON にし、読み上げも既定で ON にする。

## Key Changes
### 体験設計
- トップ画面は大きい `はじめる` ボタンと現在の到達ステージのみ表示する。
- 1ステージは 5〜10 問で構成し、終了時に `できた` 演出と星評価を出す。
- 星評価は `3段階` とし、`1星=最後まで遊べた`、`2星=ミス2回以下`、`3星=ノーミス` で判定する。
- `ミス` は `1問の中で1回以上誤キー入力があれば1ミス` と数え、同じ問題内の連続誤入力は加算しない。
- 出題は `ひらがな 1文字` または `やさしい単語` を大きく表示し、対応するローマ字を入力させる。
- 問題表示時は、ひらがなを読む力に個人差がある前提で、自動読み上げを標準動作にする。
- 正解時は効果音、簡単なアニメーション、`400ms` 後の自動遷移を入れる。
- 不正解時は強い失敗演出は避け、弱い効果音だけを鳴らし、入力文字列は進めず、次に必要なキー強調を維持する。
- ステージ進行中は「今どこまで打てたか」と「次にどのキーを押すか」を常に視覚化する。

### キーガイドUI
- プレイ画面に、英字キー中心の簡略キーボードを常設表示する。
- 現在の正解ローマ字列から、未入力先頭の1文字を `次に押すキー` として判定する。
- 対応キーだけを他キーと明確に異なる色で強調し、枠線と軽い拡大も併用して色以外でも識別できるようにする。
- 入力が1文字進むたびに、強調キーも即座に次の文字へ移る。
- ガイドが ON のときはキー強調を表示し、OFF のときは通常色のキーボードだけ表示する。
- ガイド切替は保護者向け設定画面に置き、トップ画面右上の歯車を `マウス左ボタンで2秒長押し` したときだけ開けるようにする。
- 仮想キーボードは `A-Z` のみ表示し、記号キーや修飾キーは初版では扱わない。
- 対応配列は `MacBook Pro の英字キー列` を前提にし、文字キーのみを使うことで JIS / US 差分の影響を避ける。
- 実キー入力と画面上キー表示は論理的に一致させ、クリック入力は初版では不要とする。

### 学習ルール
- 初期解放は `a/i/u/e/o`、次に `ka/ki/ku/ke/ko`、その後 `sa/su/se/so`、最後に `shi` を単独ステージとして解放する。
- 初版の正解判定は学習上の混乱を減らす代表形を表示しつつ、入力は一部の代替表記も許可する。
  例: `shi` 問題では `shi` と `si` を正解扱いにし、クリア演出では見本として `shi` を表示する。
- 出題順は固定カリキュラムベースにし、同一ステージ内で軽いランダム化を行う。
- 難所になりやすい `shi` のような例外は、通常問題投入前に見本表示とキーガイド付きの練習問題を必ず入れる。
- 単語問題は、その時点で学習済みの文字だけで組めるものに限定する。
- 単語問題は各ステージ解放ごとに `5〜10語` を用意し、`shi` 単独ステージ解放前の単語には `shi` を含めない。
- 単語問題の読み上げは `単語全体を読む -> 短い間 -> 先頭文字を読む` の順で行い、最初に意味のまとまり、その後に入力開始点を伝える。

### UI / 技術方針
- フロントエンドのみで完結する構成にする。バックエンドは持たない。
- 公開しやすさを優先して、静的ホスティング前提の SPA とする。
- 進捗保存は `localStorage` を使い、保存内容は `解放ステージ`、`各ステージ最高評価`、`ステージ挑戦回数`、`キーガイドON/OFF`、`読み上げON/OFF` に絞る。
- `ステージ挑戦回数` はステージ開始時点で `+1` し、途中離脱も挑戦として数える。用途は将来の難易度調整や保護者向け参考情報に限定し、子ども向け画面には表示しない。
- `localStorage` が使えない場合はメモリ保持にフォールバックし、そのセッションでは保存されないことを保護者向け設定画面で表示する。
- 読み上げはブラウザ標準の Web Speech API を使い、問題表示時にひらがなや簡単な指示を発話する。
- Web Speech API が使えない場合は音声なしで成立するUIにし、録音音声は初版では持たない。
- 操作は `入力欄への自動フォーカス` を基本にして、毎回クリック不要にする。
- 対応ブラウザは macOS 上の Google Chrome 最新安定版と、その直前の主要版を対象にする。
- 保護者向け設定画面には `進捗をリセット` を置き、確認ダイアログ経由で `settings` と `progress` を初期化できるようにする。
- デザインは幼児向けに、文字を大きく、情報量を少なく、押せる領域を広くする。装飾は親しみやすいが過刺激にならない方向でまとめる。

### 公開インターフェース / データ
- ステージ定義はコード内の設定配列で管理する。
- 各問題データは少なくとも `label(ひらがな表示)`, `answer(代表ローマ字)`, `acceptedAnswers(許可入力一覧)`, `voiceText(読み上げ用)` を持つ。
- キーガイド用に `answer` から現在必要な次キーを導出する共通ロジックを持つ。
- 進捗データは `typing-app/v1/settings` と `typing-app/v1/progress` のキーで保存し、将来の教材追加時はバージョン番号を上げて新キーへ移行する。
- `progress` には少なくとも `unlockedStageIds`, `bestStarsByStage`, `attemptCountByStage` を持たせる。
- 初版ではユーザーアカウント、ランキング、複数プロフィール、管理画面は入れない。

## Test Plan
- 母音ステージで、正しいキー入力ごとに進捗表示が更新され、強調キーが次の文字へ移ること。
- `ka` や `shi` のような複数字問題で、未入力先頭文字だけが常に強調されること。
- `shi` 問題で `shi` と `si` の両方が通り、結果表示では代表形 `shi` が案内されること。
- ガイド ON/OFF を切り替えた結果が保存され、Chrome 再起動後も復元されること。
- 星評価が `1星/2星/3星` の基準どおりに判定されること。
- ステージクリア後に解放状況と最高評価が保存されること。
- 正解から `400ms` 前後で次の問題へ進み、テンポが過度に遅くならないこと。
- 問題切り替え時に入力欄へ自動フォーカスし、マウス操作なしで継続できること。
- 読み上げが使える環境では問題表示時に音声が再生され、使えない環境では音声なしでもプレイ継続できること。
- MacBook Pro の Chrome で、トップ画面、プレイ画面、結果画面、仮想キーボード表示が崩れないこと。
- 誤入力時に進行不能にならず、正しい次キーの案内が維持され、入力状態が壊れないこと。
- `localStorage` が無効でもセッション中プレイは継続でき、保存不可の表示が出ること。
- 保護者設定画面が長押し操作でのみ開き、通常タップでは開かないこと。
- 進捗リセット実行後にステージ解放・星評価・設定が初期状態へ戻ること。
- 自動テストは `Vitest` でローマ字判定・進捗保存・設定フォールバックのロジックを検証し、主要導線は手動 QA で Mac Chrome 実機確認する。

## Assumptions
- 初版は日本語話者の幼児1名利用を前提にする。
- 公開先は静的ホスティングを想定し、ログインやサーバ保存は行わない。
- 音声はブラウザ依存があるため、品質より「補助として使えること」を優先し、音声なしでも成立する設計にする。
- 推奨実装は `React + Vite`。状態管理と画面遷移が素直で、静的公開もしやすいから。
- 初版では濁音、拗音、促音、長音、ん の特殊ルールは後続フェーズに回す。
- 仮想キーボードは英字キー学習補助表示であり、初版では英字以外のキー学習は対象外とする。
