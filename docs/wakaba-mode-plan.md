# わかばモード追加計画

## Summary

- 新しく「わかばモード」を追加し、幼稚園 `わかば` のクラスメイト名と担任名をタイピングさせる。
- 初期対象は `くま組` のみとし、内部設計は複数クラス対応にして、他クラスはデータ追加時にホームから自動で選べるようにする。
- UI/UX は実践モードをベースにし、`かな` の名前を大きく表示して、入力はローマ字で行う。
- 個人名は Git 管理外に置き、公開リポジトリへ実名を含めない。
- ビルド互換のため、実名データ本体とは別に `src/wakaba.local.example.ts` をコミットし、`src/wakaba.local.ts` が無い環境でも空配列で起動できる構造にする。
- 出題順は毎回ランダムにし、担任名もクラスメイトと同じ一覧に混ぜて出す。
- 結果は実践モードと同じく `ノーミス完答数` を出し、履歴もクラス別に保存できる形にする。

## Key Changes

### モード構成とホーム導線

- 専用ルートを追加する。
  - 推奨: `/wakaba`
  - 結果画面: `/wakaba/result`
- ホームに `わかばモード` セクションを追加する。
- 組選択はホームで行い、初期表示は `くま組` のみ出す。
- 将来はデータ定義にクラスを追加すれば、ホームの選択肢が自動で増える構成にする。
- 複数クラス時はクラスチップを横並びで出し、選択中クラスを切り替えてから開始ボタンを押す構成にする。
- 未実装クラスのプレースホルダは出さない。
- クラス配列が空の環境では `わかばモード` 導線自体をホームに出さない。

### 名前データの持ち方

- 名前データはクラス単位でローカル専用ファイルに持つ。
- ファイルは `src/wakaba.local.ts` のような名前で分離し、`.gitignore` 対象にする。
- リポジトリには `src/wakaba.local.example.ts` を置き、型・サンプル構造・差し替え手順だけを共有する。
- 本番 build や Vercel では `src/wakaba.local.ts` が無い前提を許容し、その場合は空クラス配列として扱う。
- ユーザーから提供されるのは `かな` の名前のみを前提にする。
- 各クラス定義は少なくとも以下を持つ。
  - `classId`
  - `classLabel`
  - `members`
- 各メンバーは少なくとも以下を持つ。
  - `id`
  - `nameKana`
- `id` は履歴の安定性を優先し、`kuma-kuroda-sayaka` のような人に紐づく固定キーをローカルファイル内で明示する。
- 初期実装では `くま組` のみ定義する。
- 担任名も `members` 配列に同列で含め、表示や採点で特別扱いしない。

### ローマ字化と入力判定

- 提供される `かな` から、アプリ側で入力用ローマ字を生成する。
- ローマ字規則はアプリ全体で使っている単一路に合わせる。
  - 例: `し=shi`, `ち=chi`, `つ=tsu`, `ふ=hu`, `ぢ=di`, `づ=zu`, `ん=n`
- 名前表示は `かな` を大きく出し、ローマ字は表示用に半角スペース区切りで見せてよい。
  - 例: `くろだ さやか` に対して表示は `kuroda sayaka`
  - 入力判定はスペースなしの `kurodasayaka`
- 入力進捗表示とキーガイドは、既存の `Question` ベースの仕組みを流用する。
- かなからローマ字への変換は、名前用に専用の変換関数またはテーブルを追加する。
- 拗音・濁音の変換規則は五十音表モードと同じ定義を参照して揃える。
- 変換表にない文字が含まれていた場合は例外を投げ、どの名前で失敗したか開発時に分かるようにする。静かにスキップしない。
- `ん` は常に `n` で固定し、今回の提供予定名では `ん + や/ゆ/よ/母音` の曖昧ケースが無い前提で進める。
- 初期実装では、提供される名前はひらがなのみで、現在のローマ字規則で一意に変換できるものを前提にする。

### プレイフローと採点

- 実践モードの UI をベースにしつつ、`PracticePlay` / `PracticeResult` の中身を汎用の単語練習コンポーネントへ抽出して共有する。
- `1回` の出題数は「そのクラスに登録されている全員分」とする。固定 `10問` にはしない。
- 出題順は毎回ランダムにする。
- 担任名は同じ出題プールに混ぜる。
- 誤答時は実践モードと同様にその名前に留まり、正しく打ち切るまで次へ進めない。
- 途中離脱時は実践モード踏襲とし、その回の履歴は保存しない。
- 結果は `ノーミス完答数 / 総人数` を表示する。
- 結果画面は実践モード風にし、`もういちど` と `トップにもどる` を出す。
- 結果画面は `location.state` ガードを持たせ、直アクセス時はトップへ戻す。

### 保存データと履歴

- 実践モードとは別に、`わかばモード` 用の履歴保存を追加する。
- 保存内容は少なくとも以下を持つ。
  - `playedAt`
  - `classId`
  - `questionCount`
  - `correctCount`
  - `memberIds`
- 意味は実践モードと揃え、`correctCount` は `ノーミス完答数` とする。
- 保存先は専用キーに分ける。
  - 例: `typing-app/v1/wakaba-log`
- 履歴は新しい順で保持し、上限件数は実践モードと同じく `20件` に揃える。
- ホームには選択中クラスの `前回` と `ベスト` を表示する。
- `ベスト` は `correctCount / questionCount` の比率が最も高い回を採用し、表示自体はその回の実値 `X / N` を使う。
- ログ `0件` 時は空状態メッセージを出す。
- `resetAllStorage()` 実行時は `わかばモード` 履歴も削除対象に含める。
- Vercel など公開環境でローカル名簿ファイルを載せない場合は、クラス配列が空のため `わかばモード` は非表示のままにする。

## Public Interfaces / Types

- 追加ルート:
  - `/wakaba`
  - `/wakaba/result`
- 追加型または定義:
  - `WakabaClassDef`
  - `WakabaMember`
  - `WakabaLogEntry`
  - `WakabaResultState`
- 追加ロジック:
  - `かな -> ローマ字` 変換関数
  - クラス別出題セット生成関数
- 既存の単語練習画面ロジックは汎用コンポーネントに抽出し、実践モードとわかばモードで共有する。
- 保存 API:
  - `loadWakabaLog()`
  - `appendWakabaLog()`
- `Question` 構造は流用し、名前データから `Question` へ変換してプレイ画面に渡す。
- 追加ファイル:
  - `src/wakaba.local.example.ts`
  - ローカル専用の `src/wakaba.local.ts` は Git 管理外

## Test Plan

- ホームに `わかばモード` 導線が追加され、`くま組` を選んで開始できること。
- `src/wakaba.local.ts` が存在しない環境でも `npm run build` が通り、ホームに `わかばモード` 導線が出ないこと。
- くま組の名前一覧から、担任を含めた全員がランダム順で出題されること。
- 問題カードは `かな` の名前を大きく表示し、ローマ字は表示用にスペース区切りで見せつつ、入力判定はスペースなしで通ること。
- 実名ではなくテスト専用のダミー名データを使って、`しゅ`・`ち`・`ん` などを含む名前が現在のローマ字規則で例外なく変換できること。
- 誤入力してもその名前に留まり、正しく打ち切るまで次へ進まないこと。
- 全員分が終わると `/wakaba/result` に遷移し、`ノーミス完答数 / 総人数` が表示されること。
- 結果画面へ `state` なしで直接アクセスした場合はトップへ戻ること。
- 結果保存時に `classId`・人数・ノーミス数・出題 ID が履歴へ追加されること。
- ホームで選択中クラスの `前回` と `ベスト` が履歴から復元されること。
- `resetAllStorage()` 後に `わかばモード` 履歴も消えること。
- `npm test` と `npm run build` が通ること。

## Assumptions

- 初期実装で提供される名前は `くま組` のみ。
- 名前データは `かな` のみ提供され、ローマ字はアプリ側で生成する。
- 個人名は公開リポジトリに含めず、ローカル専用ファイルとして管理する。
- リポジトリには実名を含まない `example` ファイルだけをコミットし、公開 Vercel 環境では `わかばモード` を空状態のまま無効化する。
- 提供される名前は、現在のローマ字規則で一意に変換できるひらがな表記を前提にする。
- 担任名はクラスメイトと同じ出題プールに混ぜ、表示上も特別扱いしない。
- 他クラスはデータ追加時に自動でホーム選択肢へ出る設計にし、今は `くま組` だけ表示する。
