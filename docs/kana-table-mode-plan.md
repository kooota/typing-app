# 五十音表モード追加とローマ字表記統一計画

## Summary

- 新しく「五十音表モード」を追加し、画面に五十音と対応ローマ字を表で常時表示しながら、`あ -> い -> う -> え -> お` のような固定順で `1文字` ずつ入力させる。
- このモードは毎回最初の文字から始め、進行の保存やログ保存は行わない。
- 五十音表は `1枚` の大きい表で表示し、基本の五十音に加えて濁音・半濁音・拗音まで載せる。
- ローマ字表記は新モードだけでなく、通常ステージ・実践モードも含めて全面的に見直すが、指定どおり `はひふへほ = ha/hi/hu/he/ho` を維持する。
- ホームでは通常ステージ一覧の下に、実践モードと並ぶ別モード導線として追加する。

## Key Changes

### 五十音表モード

- 専用ルートを追加する。
  - `/kana-table`
  - `/kana-table/result`
- 画面中央に `1枚` の大きい五十音表を置き、各マスに `かな + ローマ字` を表示する。
- 現在の対象文字だけを明確に強調し、入力済みは完了状態、未到達は通常状態で見分けられるようにする。
- 進行順は固定にする。
  - 基本五十音: `あいうえお -> かきくけこ -> ... -> わをん`
  - 濁音・半濁音: `がぎぐげご -> ざじずぜぞ -> だぢづでど -> ばびぶべぼ -> ぱぴぷぺぽ`
  - 拗音: `きゃきゅきょ -> しゃしゅしょ -> ちゃちゅちょ -> にゃにゅにょ -> ひゃひゅひょ -> みゃみゅみょ -> りゃりゅりょ -> ぎゃぎゅぎょ -> じゃじゅじょ -> びゃびゅびょ -> ぴゃぴゅぴょ`
- 拗音は `1文字` として扱い、`kya` や `sha` を最後まで打ち切るまで同じマスに留まる。
- 各文字は対応ローマ字を最後まで打ち切った時点で次の文字へ自動遷移する。
- 最後まで終えたら専用の完了画面を表示し、`もういちど` と `トップにもどる` を出す。
- このモードは毎回最初から開始し、途中再開や進捗保存は持たない。

### 入力・表示ルール

- 五十音表モードでも既存の隠し入力欄、効果音、キーガイド、読み上げの仕組みを再利用する。
- 五十音表モードのガイドは「今打つべきローマ字の次キー」に合わせて既存の仮想キーボードを光らせる。
- 表の各マスでも、現在の対象文字は `かな` と `ローマ字` の両方を強調表示する。
- 読み上げは対象文字ごとにその `かな` を読むだけにし、単語用の二段読み上げは使わない。
- 誤入力時はその文字に留まり、入力を進めない。通常ステージ同様に弱い失敗音だけ鳴らす。

### ローマ字表記の全体統一

- 五十音表モード・通常ステージ・実践モードの `answer` / `acceptedAnswers` / 表示ローマ字を同じ規則に揃える。
- 固定する表記は以下とする。
  - `し -> shi`
  - `ち -> chi`
  - `つ -> tsu`
  - `ふ -> hu`
  - `じ -> ji`
  - `ぢ -> di`
  - `づ -> zu`
  - `ん -> n`
  - `を -> wo`
  - `じゃ/じゅ/じょ -> ja/ju/jo`
  - `しゃ/しゅ/しょ -> sha/shu/sho`
  - `ちゃ/ちゅ/ちょ -> cha/chu/cho`
- `はひふへほ` は `ha/hi/hu/he/ho` を維持し、一般的表記より行の一貫性を優先する。
- 初期実装では各文字の正解は `1通り` に固定し、複数表記許可は入れない。
- `stages.ts` と `practiceWords.ts` はこの方針に合わせて全面更新する。
- 進捗互換性は `StageId` 不変により維持する。既存 `typing-app/v1/progress` の保存データ移行は不要。

### 五十音表データ構造

- 新モード用に、表表示順と入力順を兼ねる専用定義配列を追加する。
- 各要素は少なくとも以下を持つ。
  - `id`
  - `kana`
  - `romaji`
  - `section`
  - `tableRow`
  - `tableCol`
- 基本五十音・濁音/半濁音・拗音を `1画面` に載せるが、視認性のためセクション見出しは分ける。
- `Question` をそのまま流用するより、五十音表表示に必要な座標情報があるため、新モード用の表示データ定義を別で持つ。
- ただし入力判定自体は既存の `romaji.ts` ユーティリティを使い回す。

### ホーム画面導線

- ホームの通常ステージ一覧はそのまま維持する。
- 実践モードセクションの近くに、新しい `五十音ひょう` モードの開始ボタンを追加する。
- この導線には短い補助文を付ける。
  - 例: `ひらがなと ローマ字を 1つずつ じゅんばんに おぼえよう`
- 五十音表モードにはログやベスト表示を持たせない。
- 既存の実践モード導線・通常ステージ導線の並びは崩さず、第三のモードとして追加する。

## Public Interfaces / Types

- 追加ルート:
  - `/kana-table`
  - `/kana-table/result`
- 追加型または定義:
  - `KanaTableEntry`
  - 必要なら `KanaTableResultState`
- 既存変更:
  - `stages.ts` の通常教材ローマ字と既存ペア問題文字列
  - `practiceWords.ts` の実践単語ローマ字
  - 既存テストの期待値
- 保存データ:
  - 五十音表モード用の永続化キーは追加しない

## Test Plan

- ホームに五十音表モードの開始導線が追加され、押下で `/kana-table` に遷移すること。
- 五十音表モードで `あ` から始まり、`a` 入力で `い` に進むこと。
- `お` の後に `か`、基本五十音の最後の後に濁音、濁音の後に拗音へ固定順で進むこと。
- `kya` / `sha` / `cha` などの拗音は途中入力で別マスへ遷移せず、最後まで打ってから次へ進むこと。
- 現在の対象マスだけが強調され、入力済み・未到達と見分けられること。
- 誤入力しても次へ進まず、対象文字が維持されること。
- 最後の拗音まで完了すると `/kana-table/result` に遷移し、`もういちど` と `トップにもどる` が表示されること。
- 五十音表モード結果画面へ `state` なしで直接アクセスした場合はトップへ戻ること。
- 五十音表モードは再入場時に必ず最初の `あ` から始まること。
- 通常ステージと実践モードのローマ字表示・正解判定が新ルールへ統一されていること。
- 既存の `stages.test.ts`、`Play.test.tsx`、実践モード関連テストなど、訓令式前提の期待値を新ルールに更新すること。
- `npm test` と `npm run build` が通ること。

## Assumptions

- 五十音表モードは「反復練習用」であり、保存・履歴・ベスト記録は持たない。
- 拗音まで含めるため `1画面` の情報量は多くなるが、セクション分割と現在位置の強調で対応する。
- ローマ字は完全な一般的表記ではなく、指定された教材ルールを優先する。
  - `shi/chi/tsu` は採用
  - `hu`, `di`, `zu`, `n` は指定どおり固定
- 初期実装では表記ゆれ許容は入れず、表示ローマ字と入力正解を `1通り` に固定する。
